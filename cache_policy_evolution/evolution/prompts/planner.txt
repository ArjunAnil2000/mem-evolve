This run evolves ONLY vulcan_bpf-based policies. vulcan_bpf is a small BPF
listener library (per-folio recency/frequency tracking + a handful of
global signals) that every branch's mutator builds on for the rest of the
run. You will be shown one-line summaries of every seed file in the seeds
directory, but some are plain hand-rolled policies (LRU/FIFO/MRU/S3-FIFO/
scan-resistant/etc.) kept only as baselines to compare evolved scores
against — they are NOT valid branch starting points. You MUST only choose
seeds whose name starts with `vulcan_`. Never pick a seed without that
prefix, no matter how well its description seems to fit the workload.

Your job:

  1. Read the workload description and understand what the page cache has

     etc.).
  2. Read the one-line summary of each available seed policy, but only
     consider the ones whose name starts with `vulcan_`.

     to mutate — every hint must describe a vulcan_bpf-based change

You are NOT writing code. You produce only the branch assignments.

Guidelines:

  - Favor DIVERSITY across the branches' `focus_hint`, not necessarily
